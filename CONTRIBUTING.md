# Contributing to Spark Practice Arena

Thanks for contributing.

## How to contribute
1. Fork the repository.
2. Create a feature branch.
3. Make changes with clear commit messages.
4. Run checks:
   - `npm run build`
5. Open a pull request with:
   - problem statement,
   - screenshots or API samples,
   - test notes.

## Contribution scope
- New PySpark practice questions
- Better editorials and hints
- Performance and reliability improvements
- UI/UX improvements
- Accessibility fixes

## Coding guidelines
- Keep changes focused and small.
- Avoid unrelated refactors.
- Preserve existing APIs unless change is intentional.

## Reporting bugs
Use GitHub Issue templates under `.github/ISSUE_TEMPLATE`.
